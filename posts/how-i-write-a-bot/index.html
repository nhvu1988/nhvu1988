Bot Framework and LUIS

After some researching, I decide to use Bot Framework to build my chatbot since it supports nodejs which‚Äôs my favorite programming language.

I use LUIS to add natural language understanding to my bot. I compared it with api.ai and wit.ai and see it has a good design, friendly UI to define an intent, add new utterance, train and publish the application. They also have 2 videos for me can easily get the overview of LUIS üòç.

Seem everything relates to coding with bot framework, training LUIS to understand intent is easy‚Ä¶ until I have many types of Date Ranges need to be supported in my chatbot. First of all, thanks to LUIS, it has a set of pre-built entities to recognize date and time in builtin.datetime. It will understand when I say yesterday, 1st November or last week, in 3 first questions above, and will convert that to a date.

Chrono

Since the LUIS with builtin datetime did good jobs on parsing Datetime, the next requirement is how I can parse a date ranges from a specific date (or month) to another specific date (or month)? For example in my fourth question, it‚Äôs Jan - Mar 2016

Chrono supports most date and time formats, such as:

  • Today, Tomorrow, Yesterday, Last Friday, etc
  • 17 August 2016 - 19 August 2016
  • January - March 2016
  • 5 days ago

So I used it to try to recognize if LUIS couldn‚Äôt give me any information about date ranges. Two things I would like to note is I had to use every values in the knownValues to parse a date and

always parse date in UTC

function createUTCDateFromKnownValues(year, month, day, isFrom) {
    // (year, null, null, ?)
    if (year && !month && !day)
        return isFrom ? new Date(Date.UTC(year, 0, 1)) : new Date(Date.UTC(year, 11, 31, 23, 59, 59));
    // (year, month, null, ?)
    if (year && month && !day) {
        if (isFrom) return new Date(Date.UTC(year, month - 1, 1));

        var firstNextMonth = new Date(Date.UTC(year, month, 1, 23, 59, 59));
        firstNextMonth.setDate(firstNextMonth.getDate() - 1);
        return firstNextMonth;
    }
    // (year, month, day, ?)
    if (year && month && day) {
        return isFrom ? new Date(Date.UTC(year, month - 1, day)) : new Date(Date.UTC(year, month - 1, day, 23, 59, 59));
    }
}
